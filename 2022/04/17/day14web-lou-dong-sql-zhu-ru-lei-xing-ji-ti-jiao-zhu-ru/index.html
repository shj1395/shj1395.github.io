明确参数类型

  1. 数字 整型int float、double 等

  2. 字符 通过引号来给计算机提示这就是一个字符串

    要点：$name=$_get[‘x’];

     $sql=” select * from user where name=’$name’”;

    如果注入语句： ？x=xiaodi and 1=1

    数据库查询语句：select * from user where name=‘xiaodi and 1=1’;使得查询语句失效

    正确的注入语句： x=xiaodi’（需要使用单引号闭合前面的使得后面的注入语句生效） and 1=’1

  3. sql搜索 应用不同闭合方式不同 此类常用闭合方式为（% %）

    语法 select * from user where name like '%com%'

查询emails表中email_id包含com的数据

  1. JSON: 是一种格式 、JSON的语法可以表示以下三种类型的值：简单值、JSON对象和数组。闭合是去闭合SQL语句

    // 简单值
"Hello World!" // 字符串
99 // 数值
true // 布尔型
false // 布尔型
null

// 在JSON中不能使用的值
NaN // 数值不能是NaN
Infinity // 数值不能是Infinity
undefined // 在JSON也不可以使用JavaScript中的undefined
'Hello World!' // 字符串必须使用双引号表示，不能使用单引号
0x1 // 数值必须以十进制表示，不能使用十六进制

    // 对象，对象的属性名必须使用双引号，值要是字符串也必须使用双引号

  } // 这个地方也不可以有逗号，因为也是对象的最后一个属性成员
}

  5. HTTP头部

    服务器变量：$_SERVER–$_SERVER 是一个包含诸如头部(headers)、路径(paths)和脚本位置(script locations)的数组。获取当前访问者的一些信息，例如可以获取当前设备是电脑还是手机，然后返回对应的手机端或电脑端网页，这个过程会导入到数据库查询数据，存在交互，就必然存在漏洞

    注入漏洞原理：接收数据，代入数据库查询就会存在sql注入漏洞，我们通过篡改接收的数据（如果对接受的数据做了过滤的话，需要想办法让注入的语句生效，闭合等等），让服务器代入到他的数据库执行我们构建的语句查询我么想知道的数据

    （注: 在 PHP 4.1.0 及以后版本使用。之前的版本，使用 $HTTP_SERVER_VARS）
