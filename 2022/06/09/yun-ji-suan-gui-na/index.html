云计算归纳

发表于2022-06-09|更新于2022-06-09

云计算

概念：云计算是一种模型，可以实现随时随地、便捷的、按需的从可配置计算机共享池中获取所需的资源、资源可以快速供给和释放，使管理的工作量和服务提供者的介入降至最少。

分类：

  • 公有云：为广大用户提供服务–不安全
  • 私有云：企业内部–安全
  • 混合云：公有云和私有云的混合

云计算交付模式

​ Iaas(基础设施即服务)、Paas(平台即服务)、SaaS(软件即服务)（小程序在线编辑）

云计算关键技术

  1. ​ 虚拟化技术
  2. 多租户技术
  3. 并行计算技术
  4. 分布式存储技术
  5. 软件定义网络技术

云计算的特点

  1. ​ 按需自助服务
  2. 泛在接入
  3. 多租户
  4. 快速弹性收缩
  5. 服务可测量
  6. 资源池化

虚拟化

概念：把具体的物理硬件资源通过软件切割，变成更细颗粒度的资源供给给虚拟机使用。

虚拟化作用：

  1. ​ 提高资源利用率
  2. OS与硬件进行解耦
  3. 有独立于硬件的生命周期

hypervisor又称为vmm，虚拟机监视器，虚拟化软件层

hypervisor的作用

  1. ​ 将物理资源池转化为虚拟资源供虚拟机使用
  2. 将虚拟机指令调用给物理硬件

虚拟化的特点：分区、隔离、封装、独立

横向堆叠技术：把多个同种设备通过特定的链路连接起来，从逻辑上作为一个设备使用。

虚拟化分类：宿主型、裸金属型

cpu虚拟化：

  1. ​ 全虚：所有资源由vmm提供；OS指令会被hypervisor捕获、拦截翻译；VMM对上系统全虚，对下硬件hardware屏蔽隔离

  2. 半虚 （资源仍然由VMware这类软件提供,但是需要修改os内核,os必须是开源的,虚拟机如果使用特权指令，虚拟机会给vmm发送hyper call的调用，vmm通过拦截hyper call翻译特权指令，以防特权指令损坏硬件系统。）捕捉hyper call调用

  3. 硬虚 （硬件辅助虚拟化）

    非root模式：用户态发生的指令

    root模式：内核态发生的指令

    例子：当用户态发生特权指令，先发送到非root模式处理，再发到root模式处理，然后倒序返回。透传、cpu,men,I/O 都支持三种模式

kvm i/o /操作流程 默认

过程：

  1. 客户机的设备驱动程序发起 I/O 请求操作请求
  2. KVM 模块中的 I/O 操作捕获代码拦截这次 I/O 请求
  3. 经过处理后将本次 I/O 请求的信息放到 I/O 共享页 （sharing page），并通知用户空间的 QEMU 程序。
  4. QEMU 程序获得 I/O 操作的具体信息之后，交由硬件模拟代码来模拟出本次 I/O 操作。
  5. 完成之后，QEMU 将结果放回 I/O 共享页，并通知 KMV 模块中的 I/O 操作捕获代码。
  6. KVM 模块的捕获代码读取 I/O 共享页中的操作结果，并把结果放回客户机。

集群：HA、DRS、DPM

同一个集群的要求：同种架构，同种类型相同的物理配置。

HA：高可用性（业务延续不中断） 主要做迁移用，保证业务 的完整性。HA因情况而开启，如果CNA负荷重建议HA不开启。

迁移：

  1. 热迁移——业务不中断，数据不丢失，用户无感知（手动、自动）
  2. 冷迁移——业务中断，数据丢失，客户感知（手动）

触发的条件：

  1. CNA故障或意外断电（心跳线告知vrm,由vrm将vm迁移到其他正常的CNA中。）
  2. vm故障 蓝屏/死机不工作（开启HA迁移由vm中的tools工作收集vm的信息如无任何信息表明vm发生故障，tools告知vrm，由vrm对vm进行迁移）

要迁移的数据：迁移内存的数据。

迁移成功的条件：

1. 同一个集群

2. 同一个共享存储

3. 同一个DVS(Distrubuts Vitural Switch)分布式虚拟交换机

DRS（动态资源调度） ：负载均衡、在集群中找到负载最轻的主机（Distributed Resource Scheduler）分布式资源调度程序，根据对资源池资源负载的动态监控，合理触发均匀分配规则，最终实现资源池中的物理服务器之间重新分布虚拟机的目的 。

DPM（动态电源管理）

基板管理控制器（Baseboard Manager Controller，简称BMC）

vrm通过bmc开关机

网络虚拟化

CNA里面有两台交换机一台ovs（内部通信）也叫br-int，一台（负责外部通信）跟port3连接的叫br-eth，跟bond连接的叫br-bond

ovs负责业务、dvs负责管理、br-bond负责业务

vps 虚拟交换端口 ovs 虚拟交换机 uplink 上联口

1.1ping不同1.2的原因

  1. ​ sw没有配置vlan
  2. 没有配置trunk链路
  3. sw与uplink之间没有trunk链路

uplink端口

  1. ​ 与外界通信的端口
  2. 每个ovs只有一个
  3. 可物理，可逻辑
  4. 是trunk口
  5. 不用配置ip

虚拟机设备队列（VMDq）

普通网卡NIC tx(发送) rx(接收)

​ VMDq不再依赖于hypervisor虚拟机管理器对数据包进行分类和排序，因此能够创建并行队列。

单根I/O虚拟化（SR-IOV）

​ vf：具备虚拟化功能、NIC网卡具备虚拟化功能

流量

  • 管理流量：vrm到cna截止
  • 业务流量：cna到交换机（sw）截止
  • 存储流量：

安全组

​ 概述：绑定在虚机的VNIC（虚拟网卡）的集合

​ 作用：对进出虚机的流量做包过滤

  • ​ 应用到VNIC
  • 每个虚机有且只有一个安全组
  • 实现包过滤技术

（ovs和虚机连接的接口叫vsp）

​ 特性：

  1. ​ 安全组有一个默认的安全组-default
  2. 同一个安全组下面虚机可以相互通信
  3. 不同安全组不能相互通信，但是里到外可以通信。

存储

​ 集中式存储、分布式存储、raid2.0

集中：使用阵列的集中化存储来存放结构化数据

集中化存储的分类：

  1. ​ 阵列
  2. NAS

分布式存储：存储非结构化数据

优点：并发系统，速度快、扩容方便

  1. ​ 块
  2. 文件
  3. 对象

结构化数据：关系型数据库组成的表格数据

非结构化数据：类似于视频、音频类的数据

串行、并行

数据写入ssd盘称为回写，存储的数据叫做缓存。

SAN网络

IP-SAN、FC-SAN

FC光纤通道

NAS虚拟化—文件共享—应用层数据SCSI—通过TCP/IP协议—裸设备不能读写—有文件系统

SAN—块存储—CIFS/NFS—可通过TCP/IP和FC协议访问—数据可读写—无文件系统

云计算分为

  • ​ 物理资源层
  • 虚拟资源层
  • 基础设施服务层（open stack）
  • 应用层

open stack

  • ​ 软硬件的统一管理
  • 异构虚拟化的统一管理
  • 异构硬件的统一管理
  • 不同数据中心的管理

三大模式

  • ​ nova：计算
  • cinder：存储
  • neutron：网络

存储虚拟化主要作用是：将物理设备整合成资源池、屏蔽存储设备的差异化

XEN架构

1、 Domian0是一个特殊的虚拟机，其具备特殊的，足够的权限直接访问底层的硬件设备，处理所有底层的I/O请求，并与其他虚拟机进行交互通信。Domian0对外部开放一个接口，使用户可以控制整个系统。没有Domian0,Xen虚拟化管理程序无法正常使用，它是整个系统第一个加载的虚拟机。domain0有设备驱动。（申请资源）

2、 domain0有1个后端驱动。

3、 domain0随设备开启而开启。

4、 后端驱动进行分时分通道进行处理数据。

CPU虚拟化

经典虚拟化：陷入 模拟

经典虚拟化存在的问题：虚拟化漏洞（非特权）

完全虚拟化：ring0 特权指令、ring3 非特权指令

CPU QOS（服务质量）

  1. cpu资源限额
  2. cpu资源份额
  3. cpu资源预留

内存 QOS

  1. 资源预留
  2. 资源份额

内存预留：虚拟机预留的最低物理内存。预留的内存会被虚拟机独占。即，一旦内存被某个虚拟机预留，即使虚拟机实际内存使用量不超过预留量。

网络QOS

磁盘QOS：IO数据读写操作/IO上线

内存复用技术

  • 内存共享，写时复制：多台虚拟机共享数据内容相同的内存页
  • 内存置换：将外部存储虚拟成内存给虚拟机使用，将虚拟机上暂时不用的数据存放到外部存储上。系统需要使用这些数据时，再与预留在内存上的数据进行交换。
  • 内存气泡：系统主动回收虚拟机暂时不用的物理内存，分配给需要复用内存的虚拟机。内存的回收和分配均为系统动态执行，虚拟机上的应用无感知。整个物理服务器上的所有虚拟机使用的分配内存总量不能超过该服务器的物理内存总量。

内存共享

​ 优点：提高资源利用率

​ 缺点：开辟新空间、完成vmm与内存的新映射关系、写数据

内存置换

​ 缺点：不能直接从硬盘读取冷数据

内存气泡

​ 缺点：当资源占满后，无法写入数据
